# DexHand 硬件规格文档 - Hardware Specifications

## 概述

本文档描述 DexHand 五指灵巧手的硬件规格和接口定义，支持 Sim-to-Real 迁移。

## 2. 执行器规格

| 型号 | 类型 | 扭矩 | 转速 | 重量 | 接口 |

| 参数 | 值 | 单位 |

### 2.3 CAN总线配置

```
+12V ──────┬─────── +12V (所有电机)
           │
CAN_H ─────┼─────── CAN_H (黄色/橙色)
CAN_L ─────┼─────── CAN_L (白色/蓝色)
           │
GND ───────┴─────── GND (所有电机)
```

**CAN总线参数：**
- 波特率：1 Mbps
- 终端电阻：120Ω (两端各一个)
- 最大节点数：64

## 4. 通信接口

### 4.1 主控制器接口

| 接口 | 用途 | 规格 |
|------|------|------|
| USB-C | 程序下载/调试 | USB 2.0 |
| CAN | 电机控制 | 1 Mbps |
| UART | 传感器数据 | 115200 bps |
| GPIO | 紧急停止 | 3.3V TTL |

### 4.2 通信协议

```
┌─────────────────────────────────────────────────────────┐
│  CAN Message Format (Standard)                          │
├─────────────────────────────────────────────────────────┤
│  ID: 0x100 - 0x12C (Motor 0-19)                       │
│  DLC: 8 bytes                                          │
│  Data: [pos_lsb, pos_msb, vel_lsb, vel_msb, ...]      │
└─────────────────────────────────────────────────────────┘
```

### 4.3 实时性要求

| 参数 | 值 |
|------|-----|
| 控制频率 | 1000 Hz |
| 传感器采样率 | 2000 Hz |
| 通信延迟 | < 2 ms |
| 端到端延迟 | < 5 ms |

## 7. 软件接口

### 7.1 初始化序列

```python
# 伪代码
1. 初始化CAN总线 (1 Mbps)
2. 扫描所有电机ID
3. 使能所有电机
4. 设置电机模式 (位置/力矩/速度)
5. 初始化I2C传感器
6. 校准关节零点
7. 启动控制循环
```

### 7.2 控制模式

| 模式 | 命令格式 | 适用场景 |
|------|----------|----------|
| 位置控制 | `SET_POSITION(motor_id, angle)` | 精确位置 |
| 速度控制 | `SET_VELOCITY(motor_id, rpm)` | 连续运动 |
| 力矩控制 | `SET_TORQUE(motor_id, nm)` | 柔顺交互 |
| PWM控制 | `SET_PWM(pin, duty)` | 简单舵机 |

## 8. 安全规范

### 8.1 电气安全

- 工作电压：12V DC (标称)
- 过压保护：14V TVS
- 过流保护：3A 自恢复保险丝
- 反接保护：二极管

### 8.2 机械安全

- 关节限位：软件 + 物理限位块
- 紧急停止：硬件互锁
- 力矩限制：软件最大力矩限制

### 8.3 操作安全

```python
# 力矩限制示例
MAX_TORQUE = {
    'thumb': 1.5,  # Nm
    'index': 1.0,
    'middle': 1.0,
    'ring': 1.0,
    'pinky': 0.8,
}
```

## 9. 故障排除

| 故障现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 电机不响应 | CAN未连接 | 检查CAN接线 |
| 电机抖动 | 波特率不匹配 | 确认1 Mbps |
| 关节回不到零点 | 编码器故障 | 重新校准 |
| 触觉无数据 | I2C地址错误 | 检查地址跳线 |
| 力矩不足 | 电压过低 | 确认12V输入 |
